Features

  • Django Built-in background management system to facilitate the management of articles, users and other dynamic content
  • Article classification, tags, pageview statistics, and normative SEO Set up
  • User authentication system, which extends Oauth2.0 authentication on the basis of Django's own user system, and supports third-party authentication such as Weibo and Github.
  • Article comment system, cool input box special effects, support markdown syntax, secondary comment structure and reply function
  • Information reminder function, login and logout reminders, receive comments and reply reminders, information management
    •                     
  • Powerful full-text search function, just enter keywords to display articles related to the entire site
    •                     
  • RSS blog subscription function and canonical Sitemap Sitemap
    •                     
  • The caching system supported by django-redis follows the caching principle and accelerates the speed of website opening.
    •                     
  • Practical online tool to implement the interface call of online crawlers
    •                     
  • Display of friendship links and referrer sites
    •                     
  • RESTful API-style API interface

Website support

  • The front end uses Bootstrap4 + jQuery to support responsiveness; the icon uses Font Awesome
    •                     
  • Backend Python 3.5.2, Django 1.11.12, other dependencies, see requirements.txt in the source code.
    •                     
  • Background database uses MySQL, cache database uses Redis
    •                     
  • Site deployment uses Nginx + gunicorn, previously used Nginx + uwsgi
    •                     
  • bootstrap-admin is used to beautify the background management system and become a responsive interface.
    •                     
  • django-allauth, etc. are used for third-party user logins
    •                     
  • django-haystack and jieba are used to support full-text search
    •                     
  • django-redis cache
    •                     
  • Use RESTful style API interface supported by django restframework
    •                     
  • Check the website source code for other dependencies
